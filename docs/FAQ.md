# ScriptZero 常见问题解答

## 安装与环境

### Q: 安装依赖时出现错误怎么办？
A: 请确保：
1. Python 版本为 3.8 或更高
2. 网络连接正常
3. 以管理员权限运行命令提示符
4. 尝试使用国内镜像源：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple/`

### Q: 运行时提示缺少模块怎么办？
A: 重新运行安装命令：
```bash
pip install -r requirements.txt
```

## 配置问题

### Q: 如何创建配置文件？
A: 有三种方式：
1. 使用配置向导：`scriptzero config wizard`
2. 复制示例配置并修改
3. 手动创建 YAML 配置文件

### Q: 配置验证失败怎么解决？
A: 检查以下几点：
1. YAML 语法是否正确（缩进、冒号后空格）
2. 必填字段是否填写完整
3. 文件路径是否存在
4. 数值范围是否合法

## 适配器问题

### Q: 适配器无法启动怎么办？
A: 检查：
1. 配置文件中路径是否正确
2. 相关程序是否安装
3. 是否以管理员权限运行
4. 防火墙或杀毒软件是否阻止

### Q: 图像识别不准确怎么办？
A: 优化方法：
1. 提高图像模板质量
2. 调整图像识别置信度
3. 为不同界面状态准备模板
4. 确保游戏分辨率与模板匹配

## 运行问题

### Q: 启动游戏或脚本框架失败？
A: 可能原因：
1. 可执行文件路径错误
2. 缺少必要权限
3. 程序已被占用
4. 系统兼容性问题

### Q: 窗口激活失败怎么办？
A: 尝试：
1. 检查游戏是否正常启动
2. 确认窗口标题是否匹配
3. 以管理员权限运行
4. 检查是否有多个同名窗口

## 性能问题

### Q: 运行速度很慢怎么办？
A: 优化建议：
1. 减少不必要的图像识别
2. 调整检查间隔时间
3. 优化图像模板大小
4. 关闭不需要的功能

### Q: CPU 占用过高怎么办？
A: 检查：
1. 图像识别频率是否过高
2. 是否同时运行过多任务
3. 系统资源是否充足

## 权限问题

### Q: 拒绝访问错误如何解决？
A: 确保：
1. 以管理员身份运行脚本
2. 相关游戏和程序也有相应权限
3. 检查 Windows UAC 设置
4. 防病毒软件未阻止操作

## 图像模板

### Q: 如何制作高质量的图像模板？
A: 制作要点：
1. 选择对比度高的元素
2. 避免复杂背景
3. 保持模板清晰
4. 为不同状态准备模板

### Q: 模板文件应该放在哪里？
A: 默认路径是 `templates/` 目录，也可以在配置中指定其他路径。

## 故障排查

### Q: 如何启用调试模式？
A: 两种方式：
1. 在配置文件中设置 `debug_mode: true`
2. 使用命令行参数：`scriptzero start --debug config.yaml`

### Q: 日志文件在哪里？
A: 默认位置在配置文件中指定，通常是 `logs/app.log`。

### Q: 如何查看详细错误信息？
A: 启用 DEBUG 级别日志或使用调试模式运行。

## 安全问题

### Q: 使用此工具是否安全？
A: 
1. 本工具仅模拟用户操作，不修改游戏文件
2. 遵守游戏使用条款
3. 建议合理使用，避免过于频繁的操作

### Q: 如何避免账号风险？
A: 建议：
1. 合理设置操作间隔
2. 不要长时间连续运行
3. 遵守游戏规则

## 扩展功能

### Q: 如何添加新的适配器？
A: 参考开发者指南，继承 BaseAdapter 类并实现必要方法。

### Q: 支持哪些游戏？
A: 目前主要支持原神，可通过扩展适配器支持其他游戏。

## 更新与维护

### Q: 如何更新到最新版本？
A: 
```bash
git pull origin main
pip install -r requirements.txt
```

### Q: 配置文件格式变更怎么办？
A: 检查更新日志，按说明迁移配置，或使用配置向导重新生成。
